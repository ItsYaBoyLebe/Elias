# knowledge/ — shared, committed reference material

This is the "technology home base" knowledge area. Files here **are**
committed to the repo, so only put things that are safe to be versioned
and (potentially) seen by anyone with repo access.

Good fits:
- How-to notes: how to deploy the site, how to add a new machine
- Machine spec sheets, manuals, supplier info that isn't confidential
- Architecture notes, decisions, diagrams
- Anything you and your brother both want to refer back to

**Not** for: customer data, pricing/margins, passwords, contracts.
Those go in `private/` (local-only) or `.env` (secrets). See
`docs/security.md`.
